Heat Dissipation: Sequential vs Parallel Performance

Purpose of the Project

This project evaluates the runtime advantage of parallel computing over sequential execution for a 2D heat dissipation simulation.

The same numerical model is implemented in two versions:

  • a sequential solver (seq_main.cc)
  • an OpenMP-parallel solver (par_main.cc)

Benchmark scripts collect timings over multiple matrix sizes and repetitions, then compare execution time and scalability.

Physical Model

The simulated system is a square plate with two fixed heat sources. Temperature is represented by a scalar field T(i,j) discretized on a 2D grid.

At each time step, internal cells are updated with a 5-point finite-difference diffusion stencil (explicit time integration).

Model parameters:

  • alpha is the thermal diffusion coefficient
  • dt is the time step

For the full mathematical derivation and validation details, see the measurement report: HD_PAOLO_ROSSI.pdf

Boundary cells are updated by mirroring adjacent internal values, approximating a zero normal gradient (insulated border). Two source cells are reset every iteration to fixed temperatures.
